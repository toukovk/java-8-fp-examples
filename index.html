# FP with Java 8 - 101
## Why should I care? Java for FP? * Java (even 8) is not Scala, Clojure, Haskell or [your-favourite]. * Every now and then Java might still be required, e.g. legacy codebase, customer requirement, ... * If so, Java 8's FP support is actually pretty nice -> if using Java, Java 8 is a big improvement.
## What to expect (from this intro)? * A couple of examples to present Java 8's new FP-enabling features. * (Basic Java & FP knowledge is assumed)

Short precap

See JavaPreJava8Comparison.java

The main things

  • Lambda expressions - functions closer to being first class sitizens
  • Stream API etc. - uniform way to work in a FPish way with collections etc.
  • Misc other smaller imprevements (Optional, default methods in interfaces, ...)
